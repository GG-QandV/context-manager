Що таке Context Manager?

Context Manager — це банк пам'яті для AI-асистентів на кшталт Claude, ChatGPT та інших.

Яку проблему він вирішує: Коли ви говорите з Claude сьогодні, він пам'ятає цю розмову. Але завтра він починає з нуля — забуває все, що ви обговорювали вчора.

Що робить Context Manager: Він зберігає ваші розмови, рішення та важливу інформацію. Коли ви починаєте нову розмову, ваш AI-асистент може переглянути, що обговорювалося раніше.

Уявіть собі таке: Context Manager — це як записник, який може читати ваш AI-асистент. Ви пишете в нього (автоматично), а ваш аsistеnt читає, коли ви починаєте нову розмову.

Що потрібно перед встановленням

Перш ніж встановити Context Manager, переконайтеся, що у вас є:

  1. Windows 10 або Windows 11 (встановлювач не працює на старіших версіях)
  2. Інтернет-з'єднання (для завантаження встановлювача)
  3. Близько 5 хвилин вашого часу
  4. Права адміністратора на вашому комп'ютері (вам потрібно буде натиснути "Так", коли Windows попросить дозволу)

Це все! Вам не потрібно встановлювати щось інше спочатку.

Встановлення на Windows 10/11

Крок 1: Завантажте інсталятор

  1. Відкрийте браузер і перейдіть на: github.com/GG-QandV/context-manager/releases
  2. Завантажте останній context-manager-setup.exe
  3. Двічі клікніть на завантажений файл
  4. Натисніть Так, якщо Windows запитає "Дозволити цій програмі вносити зміни?"

Крок 2: Слідуйте за майстром налаштування

Інсталятор проведе вас через налаштування. Він буде:

  • Встановлювати PostgreSQL (база даних) — займає близько 1 хвилини
  • Встановлювати Qdrant (пошукова система) — займає близько 30 секунд
  • Встановлювати сервіси Context Manager — займає близько 2 хвилин
  • Завантажувати AI-модель (multilingual-e5-small) — займає близько 1 хвилини
  • Запускати всі сервіси та показувати трей-іконку — займає близько 30 секунд

Загальний час: 5-10 хвилин залежно від швидкості вашого інтернету.

Крок 3: Завершення

Коли інсталятор покаже "Setup Complete", все встановлено та працює. Ви побачите кольорову іконку-коло в системному треї (нижній правий кут біля годинника).

Перевірка чи все працює

Щоб перевірити, чи Context Manager працює належним чином:

  1. Відкрийте будь-який веб-браузер (Chrome, Firefox, Edge тощо)

  2. Напишіть у адресному рядку та натисніть Enter:

http://localhost:3847/health

  1. Ви повинні побачити щось подібне:

json {"status":"healthy","postgresql":"connected","qdrant":"connected"}

Якщо ви бачите "status":"healthy" — все працює!

Якщо ви бачите помилку або сторінка не завантажується:

  • Зачекайте 1 хвилину та спробуйте ще раз (сервіси можуть ще запускатися)
  • Перегляньте розділ усунення несправностей нижче

Підключення до Claude Desktop

Крок 1: Відкрийте PowerShell від імені адміністратора

  1. Натисніть кнопку "Пуск" (логотип Windows у лівому нижньому куті)
  2. Напишіть powershell
  3. Клікніть правою кнопкою миші на Windows PowerShell у результатах пошуку
  4. Натисніть "Запуск від імені адміністратора"
  5. Натисніть Так, якщо Windows запитає "Дозволити цій програмі вносити зміни?"

Крок 2: Запустіть генератор конфігурації

Напишіть відповідну команду та натисніть Enter:

Якщо ви використовували .exe інсталятор (рекомендовано):

cd "C:\Program Files\Context Manager"
.\bin\node.exe scripts\init-mcp-config.mjs

Якщо ви використовували PowerShell скрипт (install-native.ps1):

cd C:\context-manager
node scripts/init-mcp-config.mjs

Ви повинні побачити: "MCP config written to..." — це означає, що все вдалося.

Крок 3: Перезапустіть Claude Desktop

  1. Повністю закрийте Claude Desktop (клікніть правою кнопкою миші на іконку у системному лотку → Вийти)
  2. Знову відкрийте Claude Desktop

Крок 4: Перевірте

Почніть нову розмову в Claude та запитайте:

"Чи можеш ти перевірити свою пам'ять за допомогою cm_stats?"

Claude повинен побачити Context Manager та показати вам статистику збереженого контексту.

Підключення до Cursor / VS Code

Крок 1: Відкрийте конфігурацію MCP

  1. Відкрийте Провідник
  2. Перейдіть до папки встановлення:
  3. .exe інсталятор: C:\Program Files\Context Manager\app\
  4. PowerShell скрипт: C:\context-manager\
  5. Відкрийте файл mcp.json за допомогою Блокнота

Крок 2: Скопіюйте вміст

Виділіть весь текст (Ctrl+A) та скопіюйте його (Ctrl+C).

Крок 3: Додайте до вашого IDE

Для Cursor:

  1. Відкрийте Cursor
  2. Натисніть Ctrl+Shift+P
  3. Напишіть "MCP" та виберіть "MCP: Open Configuration"
  4. Вставте скопійований вміст та збережіть

Для VS Code з Continue:

  1. Відкрийте VS Code
  2. Натисніть Ctrl+Shift+P
  3. Напишіть "Continue" та виберіть "Continue: Open config.json"
  4. Додайте конфігурацію MCP-сервера зі скопійованого вмісту

Щоденне використання Context Manager

Після встановлення вам не потрібно робити щось особливе. Context Manager працює у фоновому режимі.

Що відбувається автоматично

  • Коли ви говорите з Claude (або іншими AI-інструментами), важлива інформація зберігається
  • Коли ви починаєте нову розмову, ваш AI може переглянути, що обговорювалося раніше
  • Все зберігається локально на вашому комп'ютері — нічого не потрапляє в хмари

Що ви можете зробити вручну

Якщо ви хочете перевірити, що збережено:

  1. Відкрийте браузер
  2. Перейдіть до http://localhost:3847/health — перевірте чи все працює
  3. Перейдіть до http://localhost:3847/api/context/stats — перегляньте скільки контексту збережено

Що вам може бути цікаво

  • Де зберігаються мої дані? У базі даних PostgreSQL (дані в C:\ProgramData\Context Manager\ для .exe інсталятора, або C:\context-manager\data\ для PowerShell скрипта)
  • Чи можу я зробити резервну копію даних? Так, ви можете експортувати контекст через API
  • Скільки місця це займає? Зазвичай кілька мегабайт на розмову
  • Чи уповільнює це мій комп'ютер? Ні, він використовує дуже мало пам'яті в режимі очікування

Іконка в системному треї

Context Manager додає індикатор стану в системний трей (нижній правий кут екрану біля годинника). Індикатор — це кольорове коло, яке показує стан Context Manager одним поглядом.

Як встановлюється трей-іконка

  • На Windows: Встановлювач реєструє трей-іконку як службу. Вона запускається автоматично при вході в систему.
  • На Linux: Трей-іконка запускається разом з API Context Manager. Вона з'являється як AppIndicator у вашому робочому середовищі.
  • На macOS: Трей-іконка використовує нативний індикатор панелі меню (потрібен PyQt6).

Якщо трей-іконка не з'явилася автоматично, ви можете запустити її вручну:

Якщо ви використовували .exe інсталятор (рекомендовано):

# Windows (PowerShell від Адміністратора)
cd "C:\Program Files\Context Manager\embed\.venv\Scripts"
.\pythonw.exe -m cm_integration.tray_pyqt

Якщо ви використовували PowerShell скрипт (install-native.ps1):

# Windows (PowerShell від Адміністратора)
cd C:\context-manager
python -m cm_integration.tray_pyqt

Linux / macOS:

python -m cm_integration.tray_pyqt

Що означають кольори

Трей-іконка використовує п'ять кольорів для відображення поточного стану системи:

Колір Стан Значення Що робити
🔵 Синій Очікування Всі сервіси здорові, активної роботи немає Нічого — все добре
🟢 Зелений Активний AI-модель обробляє дані (активність за останні 5 сек) Зачекайте — нормальна робота
🩵 Бірюзовий Підключено Недавня API-активність (5–30 секунд тому) Нормальний стан після активного використання
🟡 Жовтий Попередження Node.js MCP-адаптер не працює (Fastify API ще працює) Перевірте http://localhost:8770/health
🔴 Червоний Помилка API Context Manager не відповідає Перезапустіть сервіси (див. Зупинка та запуск) або перевірте логи

Як користуватися трей-іконкою

  1. Клікніть правою кнопкою миші на кольорове коло в системному треї
  2. З'явиться меню з наступними опціями:
Пункт меню Іконка Що робить
Status ℹ️ info-thin.svg Показує спливаюче вікно з поточним станом системи
Tunnel 🌐 globe-simple-thin.svg Відкриває підменю керування тунелем (див. нижче)
Quit ⏻ power-thin.svg Закриває трей-іконку (сервіси продовжують працювати)

Керування тунелем

Тунель дозволяє зовнішнім AI-інструментам (Claude, Perplexity, ChatGPT, Grok) безпечно підключатися до вашого Context Manager через інтернет. Він використовує SSH-тунелювання (Serveo) для створення публічної URL-адреси, яка вказує на ваш локальний Context Manager.

Встановлення тунелю

Тунель вбудований у Context Manager — окреме встановлення не потрібне. Проте OAuth-адаптер (який керує автентифікацією для зовнішніх підключень) запускається як окремий фоновий процес. Щоб встановити або перевірити компоненти тунелю:

Windows (встановлюється автоматично):

  • python -m cm_integration.tunnel_manager реєструється як фоновий процес
  • Трей-іконка керує ним через своє меню

Linux / macOS (перше налаштування):

# Переконайтеся, що SSH доступний (потрібен для Serveo тунелю)
which ssh || sudo apt install openssh-client

# Запустіть тунельний адаптер вручну
python -m cm_integration.tunnel_manager

Використання тунелю з трей-іконки

Крок 1 — Запуск тунелю:

  1. Клікніть правою кнопкою на трей-іконку → Tunnel → ▶️ Start Tunnel (іконка play-thin.svg)
  2. Зачекайте кілька секунд — трей покаже сповіщення "🌐 Tunnel Ready" з URL тунелю
  3. Статус тунелю зміниться на ACTIVE з URL у підказці

Крок 2 — Копіювання даних підключення для AI-інструментів:

  1. Клікніть правою кнопкою на трей-іконку → Tunnel
  2. Ви побачите список налаштованих сервісів (Perplexity, Claude, Grok тощо), кожен з власною іконкою
  3. Наведіть на сервіс → натисніть 📋 Copy URL (copy-thin.svg) — повний URL тунелю для цього сервісу скопіюється в буфер обміну
  4. Якщо сервіс вимагає автентифікації, також натисніть 🔑 Copy Token (key-thin.svg) — токен авторизації скопіюється
  5. Вставте ці дані в файл налаштувань MCP вашого AI-інструменту

Крок 3 — Зупинка тунелю:

  • Клікніть правою кнопкою на трей-іконку → Tunnel → ⏹️ Stop Tunnel (stop-thin.svg)

Крок 4 — Перезапуск тунелю (якщо щось не працює):

  • Клікніть правою кнопкою на трей-іконку → Tunnel → 🔄 Restart Tunnel (arrows-clockwise-thin.svg)

Аварійне завершення (якщо тунель застряг):

  • Клікніть правою кнопкою на трей-іконку → Tunnel → 🗑️ Force Kill Tunnel (trash-thin.svg)

Порада: Тунель залишається працювати, навіть після закриття трей-іконки. Ви завжди можете перевірити його стан, відкривши трей знову та навівши на меню Tunnel.

Індикатори стану тунелю

Статус Значення
ACTIVE Тунель працює, зовнішні підключення дозволені
Starting… Тунель ініціалізується (зачекайте кілька секунд)
Off Тунель не запущено

Вирішення проблем з тунелем

Проблема Рішення
Тунель не запускається Перевірте, чи встановлено SSH. На Windows: Налаштування → Додаткові компоненти → OpenSSH Client.
"OAuth adapter did not respond" Порт 8769 може бути зайнятий. Опція Force Kill очищає всі процеси, що використовують цей порт.
Тунель запущено, але AI-інструменти не підключаються Спробуйте Restart Tunnel — це перестворює SSH-тунель з новим з'єднанням.
"Address already in use" Використайте Force Kill Tunnel для очищення залишкових процесів, потім спробуйте Start знову.
З'єднання переривається через деякий час Безкоштовні тунелі Serveo можуть закінчуватися після 30 хвилин бездіяльності. Просто зробіть Restart Tunnel для перепідключення.

Підключення зовнішніх AI-сервісів через тунель

Коли тунель активний, ви можете підключати до Context Manager різні AI-сервіси через інтернет. Кожен сервіс має свій URL та спосіб авторизації.

Як отримати URL та токен:

  1. Клікніть правою кнопкою на трей-іконку → Tunnel
  2. Наведіть на потрібний сервіс → 📋 Copy URL — URL скопіюється в буфер
  3. Якщо сервіс потребує токен, також натисніть 🔑 Copy Token

Perplexity

URL:  https://<ваш-тунель>/mcp
Тип:  Streamable HTTP
Токен: не потрібен

Налаштування:

  1. Відкрийте Perplexity → Settings → AI Plugins → MCP URL
  2. Вставте скопійований URL (шлях /mcp)
  3. Тип підключення: Streamable HTTP
  4. Збережіть

Claude.ai (веб-версія)

URL:  https://<ваш-тунель>/sse
Auth: Bearer Token (або OAuth)

Налаштування:

  1. Відкрийте claude.ai → Settings → Integrations → Add Custom Connector
  2. Вставте скопійований URL (шлях /sse)
  3. Якщо запитає токен — натисніть 🔑 Copy Token в треї та вставте його
  4. Збережіть

ChatGPT

URL:  https://<ваш-тунель>/mcp/chatgpt
Auth: OAuth (автоматично)

Налаштування:

  1. Відкрийте ChatGPT → Settings → Connectors → Add
  2. Вставте скопійований URL (шлях /mcp/chatgpt)
  3. ChatGPT автоматично виконає OAuth — слідуйте підказкам браузера
  4. Збережіть

Grok (xAI)

URL:  https://<ваш-тунель>/mcp/grok
Тип:  HTTP / MCP
Auth: Bearer Token

Налаштування:

  1. Відкрийте Grok → Settings → Connected Services → Add MCP Server
  2. Вставте скопійований URL (шлях /mcp/grok)
  3. Натисніть 🔑 Copy Token в треї та вставте його
  4. Збережіть

Порада: Після запуску тунелю URL змінюється при кожному перезапуску. Оновлюйте URL в налаштуваннях сервісу після перезапуску тунелю.

Зупинка та запуск сервісів

Зупинити всі сервіси

Якщо ви використовували PowerShell скрипт (install-native.ps1): Двічі клікніть на файл:

C:\context-manager\cm-off.bat

Або у PowerShell:

C:\context-manager\cm-off.bat

Якщо ви використовували .exe інсталятор: Відкрийте PowerShell від імені адміністратора та запустіть:

nssm stop cm-watchdog
nssm stop cm-mcp
nssm stop cm-api
nssm stop cm-embed
nssm stop cm-qdrant

Запустити всі сервіси

Відкрийте PowerShell від імені адміністратора та запустіть:

nssm start cm-qdrant
nssm start cm-embed
nssm start cm-api
nssm start cm-mcp
nssm start cm-watchdog

Перезапустити всі сервіси

Якщо ви використовували PowerShell скрипт (install-native.ps1): Двічі клікніть на файл:

C:\context-manager\cm-restart.bat

Якщо ви використовували .exe інсталятор: Виконайте команди зупинки вище, потім команди запуску.

Перевірити чи сервіси працюють

У PowerShell:

Get-Service cm-*, cm-qdrant, cm-embed

Ви повинні побачити всі сервіси зі статусом "Running".

Допомога та логи

Якщо щось пішло не так

  1. Перевірте стан: Відкрийте браузер, перейдіть до http://localhost:3847/health
  2. Перевірте логи:
  3. .exe інсталятор: C:\ProgramData\Context Manager\logs\
  4. PowerShell скрипт: C:\ProgramData\nssm\logs\
  5. Шукайте помилки: Відкрийте найновіший файл .log за допомогою Блокнота

Розташування файлів логів

Розташування файлу конфігурації

Ваша конфігурація знаходиться за адресою:

  • .exe інсталятор: C:\ProgramData\Context Manager\app\.env
  • PowerShell скрипт: C:\context-manager\.env

Ви можете редагувати його за допомогою Блокнота, якщо вам потрібно змінити налаштування (наприклад, пароль бази даних).

Часті запитання

Питання: "Я отримую помилку під час встановлення"

Відповідь: Інсталятор покаже спливаюче вікно з конкретною помилкою. Найпоширеніші причини:

  • Антивірус блокує інсталятор — Тимчасово вимкніть антивірус, потім запустіть інсталятор знову.
  • Недостатньо прав — Переконайтеся, що ви клацнули правою кнопкою на інсталяторі та вибрали "Запуск від імені адміністратора".
  • Інтернет-з'єднання — Інсталятору потрібно завантажити PostgreSQL та ONNX модель.

Питання: "Сервіси не запускаються після перезавантаження комп'ютера"

Відповідь: Зачекайте 30-60 секунд після завантаження. Сервіси запускаються автоматично, але завантаження моделі ONNX займає час. Якщо вони все ще не запускаються:

  1. Відкрийте PowerShell від імені адміністратора
  2. Запустіть: Get-Service cm-*
  3. Якщо якісь показують "Stopped", запустіть їх вручну: nssm start cm-api

Питання: "Claude Desktop не бачить Context Manager"

Відповідь:

  1. Переконайтеся, що ви запустили генератор конфігурації MCP (див. Підключення до Claude Desktop)
  2. Повністю закрийте Claude Desktop (клікніть правою кнопкою → Вийти)
  3. Знову відкрийте Claude Desktop
  4. Перевірте: http://localhost:3847/health повинен показувати "healthy"

Питання: "Чи можу я використовувати Context Manager з кількома AI-інструментами?"

Відповідь: Так! Ви можете підключити Claude Desktop, Cursor та інші інструменти з підтримкою MCP одночасно. Вони всі ділять один контекст.

Питання: "Чи мої дані відправляються в хмару?"

Відповідь: Ні. Все залишається на вашому комп'ютері. Context Manager на 100% локальний.

Питання: "Як видалити Context Manager?"

Відповідь:

Якщо ви використовували .exe інсталятор:

  1. Відкрийте PowerShell від імені адміністратора та запустіть:

powershell nssm stop cm-watchdog nssm stop cm-mcp nssm stop cm-api nssm stop cm-embed nssm stop cm-qdrant nssm remove cm-watchdog confirm nssm remove cm-mcp confirm nssm remove cm-api confirm nssm remove cm-embed confirm nssm remove cm-qdrant confirm

  1. Видаліть папку: C:\Program Files\Context Manager\

  2. Видаліть папку з даними: C:\ProgramData\Context Manager\

  3. (Необов'язково) Видаліть PostgreSQL та Qdrant через Налаштування Windows → Програми

Якщо ви використовували PowerShell скрипт (install-native.ps1):

  1. Зупиніть всі сервіси: двічі клікніть C:\context-manager\cm-off.bat

  2. Видаліть ці папки: C:\context-manager\ та C:\qdrant\

  3. Відкрийте PowerShell від імені адміністратора та запустіть:

powershell nssm remove cm-api confirm nssm remove cm-embed confirm nssm remove cm-mcp confirm nssm remove cm-qdrant confirm nssm remove cm-watchdog confirm

  1. (Необов'язково) Видаліть PostgreSQL та Qdrant через Налаштування Windows → Програми

Питання: "Мій антивірус блокує встановлювач"

Відповідь: .exe інсталятор підписаний і не повинен блокуватися більшістю антивірусів. Якщо ваш антивірус блокує завантаження, тимчасово вимкніть його, завантажте інсталятор, потім увімкніть антивірус знову.

Питання: "Мені потрібна додаткова допомога"

Відповідь: Відкрийте issue на GitHub: github.com/GG-QandV/context-manager/issues
