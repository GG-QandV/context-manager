What is Context Manager?

A tool that gives AI assistants memory — like a notebook where they write down what they learned and look it up later.

The Problem

Every time you talk to an AI, it starts from scratch. It doesn't remember what you discussed yesterday, last week, or even five minutes ago. All context is lost.

Every AI conversation starts from scratch — past context is lost.

The Solution

Context Manager saves conversations in two ways: as written text in a database, and as "meaning fingerprints" — numbers that represent what the conversation is about.

Saved in two formats: text (for exact search) and "digital meaning fingerprint" (for finding similar topics)

The Result

AI assistants remember past conversations, find relevant information by meaning, and share context between different AI tools — all without human intervention.

AI assistants remember past conversations and can share information with each other
Important note: Context Manager uses a standard connection method called MCP. Think of MCP as a universal plug — just like any headphones with a 3.5mm jack work with any phone, any AI assistant that supports MCP can connect to Context Manager. If your AI doesn't support MCP (does not support it yet), Context Manager also works through a regular web address (regular web address / HTTP). No special setup needed. MCP = Model Context Protocol (model context protocol) — but remembering this name is not required. It is enough to know it is the standard "communication language" between AI and external programs.
002

The system has three logical layers. Information flows from top to bottom and back.

How to read this diagram: Information flows in numbered steps (1 → 2 → 3). Layer 1 — the AI tools you use (Antigravity, VS Code, etc.). Layer 2 — Context Manager's "brain" that processes requests. Layer 3 — where data lives: • PostgreSQL — stores the conversation as readable text (like a document folder). ONNX Embedder — converts text into a mathematical "meaning fingerprint". • Qdrant — stores these "meaning fingerprints" for fast search by meaning (search by meaning, not by words). Key point: the two databases are synchronized. When you save a conversation, the text goes to PostgreSQL, then the Embedder converts it to a vector and stores it in Qdrant automatically. Both databases always contain the same data — just in different formats (text vs. meaning fingerprint). You can search in either one and get the same results.

Context Manager offers several tools — each does one specific job, like "save a note" or "search memories".

Saving Context

Three levels of detail, depending on how much you want to keep:

✓
Brief — auto-summary of the conversation in 200-300 characters. Like a tweet about what happened.
✓
Important — structured notes with topics, up to 3,000 characters. Like a meeting summary.

Search & Analysis (Search & Analysis)

Find past conversations in different ways:

✓
Semantic search — semantic search. Find conversations by meaning, even if you don't remember the exact words. For example, searching "server issues" will find conversations about "database connection problems".
✓
Filtered query — filter by parameters. Search with filters: by date, by AI agent, by save mode.
✓
Cross-agent search — cross-agent search. One AI assistant can look up what another AI assistant wrote down.

✓

✓
Statistics — statistics. Shows details for a specific assistant or session.

Connection Methods

Three ways to connect Context Manager to your AI tools:

✓
Standard MCP — for AI tools on your computer (via command line). The most common method.
✓
Web MCP — for AI tools in the browser. Uses port :8770.
✓
Direct HTTP — without MCP, via direct HTTP on port :3847. Use this if your AI does not support MCP.
What is MCP? MCP — a universal connection standard (unified communication language) for AI assistants. Imagine that every AI assistant has a "plug", and Context Manager has a "socket". MCP is an international standard that all manufacturers follow, so the plug fits the socket. Thanks to MCP, you do not need to configure each program separately — if the AI supports MCP, it automatically "sees" Context Manager and can read from and write to it. Alternative (if your AI does not support MCP): Context Manager also works via HTTP (regular web protocol — the same one websites use). In this case, the connection is configured once using a web address (URL) and password.
