001

Що таке Context Manager?

Інструмент, який дає AI-помічникам пам'ять — ніби записник, куди вони записують те, що дізналися, а потім можуть знайти це знову.

Проблема

Кожного разу, коли ви говорите з AI, він починає з нуля. Він не пам'ятає, що ви обговорювали вчора, минулого тижня — навіть п'ять хвилин тому. Весь контекст втрачається.

Кожна розмова з AI починається "з чистого аркуша" — попередній контекст губиться

Рішення

Context Manager зберігає розмови двома способами: як звичайний текст у базі даних (PostgreSQL), і як "цифрові відбитки сенсу" — числа, які показують, про що саме йшлося.

Збереження у двох форматах: текст (для точного пошуку) і "цифровий відбиток змісту" (для пошуку за схожими темами)

Результат

AI-помічники пам'ятають минулі розмови, знаходять потрібну інформацію за змістом і можуть обмінюватися контекстом між собою — без участі людини.

AI-помічники пам'ятають попередні розмови та можуть обмінюватися інформацією один з одним
Важливе пояснення: Context Manager використовує стандартний спосіб підключення під назвою MCP. Уявіть MCP як універсальний роз'єм (стандарт підключення) — так само як будь-які навушники з 3.5mm jack підходять до будь-якого телефона, будь-який AI-помічник, який підтримує MCP, може підключитися до Context Manager. Якщо ваш AI не підтримує MCP, Context Manager також працює через звичайну веб-адресу (HTTP — протокол передачі даних, за яким працюють сайти).
002

Як це працює

Система має три логічні шари. Інформація рухається згори донизу та у зворотному напрямку.

Як читати цю схему: Інформація рухається за кроками (1 → 2 → 3). Шар 1 — інструменти AI, які ви використовуєте (Antigravity, VS Code тощо). Шар 2 — "мозок" Context Manager, який обробляє запити. Шар 3 — місце, де живуть дані: • PostgreSQL (реляційна БД — дані в таблицях) — зберігає розмову як текст. • TEI Embedder (перетворювач тексту на вектор сенсу) — конвертує текст у математичний "відбиток сенсу". • Qdrant (векторна БД для пошуку за змістом) — зберігає ці "відбитки сенсу" для швидкого пошуку за змістом. Ключовий момент: обидві бази даних синхронізовані. Коли ви зберігаєте розмову, текст потрапляє в PostgreSQL, потім TEI Embedder конвертує його у вектор і зберігає в Qdrant автоматично. Обидві бази завжди містять одні й ті самі дані — просто в різних форматах (текст vs цифровий відбиток сенсу).
003

Можливості

Context Manager пропонує кілька інструментів — кожен виконує одну конкретну задачу, ніби кнопки на пульті дистанційного керування.

Збереження контексту

Три рівні деталізації, залежно від того, скільки ви хочете зберегти:

✓
Brief (коротко) — автоматичний підсумок розмови на 200-300 символів. Ніби твіт про те, що сталося.
✓
Important (важливе) — структуровані нотатки з темами, до 3 000 символів. Ніби підсумок наради.
✓
Full (повністю) — повний лог розмови дослівно. Ніби зберегти весь запис наради.

Пошук та аналіз

Знаходьте минулі розмови різними способами:

✓
Semantic search (пошук за змістом) — знаходить розмови за сенсом, навіть якщо ви не пам'ятаєте точних слів. Наприклад, пошук "проблеми з сервером" знайде розмови про "помилки підключення до бази даних".
✓
Filtered query (фільтр за параметрами) — пошук з фільтрами: за датою, AI-агентом, режимом збереження.
✓
Cross-agent search (пошук між агентами) — один AI-помічник може знайти те, що записав інший AI-помічник.

Моніторинг

Перегляд того, що знаходиться в системі пам'яті:

✓
Agent list (список агентів) — показує всіх AI-помічників та кількість збережених ними записів.
✓
Statistics (статистика) — деталі про конкретного помічника або сесію.
✓
Export (експорт) — вивантаження сесії у JSON (текстовий формат, зрозумілий програмам).

Вбудовані помічники

Розумні інструменти, які працюють автоматично:

✓
Content Detector (визначення типу вмісту) — автоматично розпізнає, що саме читає: код, команди, таблиці чи помилки — і обробляє відповідно.
✓
Time Parser (розпізнавання дат) — розуміє фрази "вчора", "останні 3 дні" українською та англійською.
✓
Chunking (розділення тексту на частини) — розбиває довгі тексти на менші шматки для кращої якості пошуку.

Способи підключення

Три способи під'єднати Context Manager до ваших AI-інструментів:

✓
Standard MCP (стандартний) — для AI-програм на вашому комп'ютері. Найпоширеніший спосіб.
✓
Web MCP (через інтернет) — для AI-програм у браузері. Використовує порт :8770.
✓
Direct HTTP (прямий доступ) — без MCP, через веб-запити на порт :3847. Якщо ваш AI не підтримує MCP.
Що таке MCP? MCP — це єдиний стандарт підключення (мова спілкування) для AI-помічників. Уявіть, що в кожного AI-помічника є "вилка", а в Context Manager — "розетка". MCP — це міжнародний стандарт, якого дотримуються всі виробники, щоб вилка підійшла до розетки. Завдяки MCP вам не потрібно налаштовувати кожну програму окремо — якщо AI підтримує MCP, він автоматично "бачить" Context Manager і може в нього писати та читати. Альтернатива (якщо ваш AI не підтримує MCP): Context Manager також працює через HTTP (протокол передачі даних — той самий, за яким працюють веб-сайти). У цьому випадку підключення налаштовується один раз через веб-адресу (URL) та пароль.

Технології

Сучасні, легкі технології. Жодних важких фреймворків — працює офлайн, на будь-якому комп'ютері.

Середовище виконання (Runtime)

Node.jsСередовище для JavaScript — двигун, який запускає сервер
TypeScriptМова програмування — JavaScript із типами для надійності
FastifyВеб-фреймворк — обробляє HTTP-запити (інтернет-звернення)
PinoЖурнал подій (логер) — записує, що робить система

Сховища (Storage)

PostgreSQLРеляційна БД (база даних) — зберігає текст з індексами для пошуку
QdrantВекторна БД — зберігає "відбитки сенсу" для пошуку за змістом
TEIСервер перетворення тексту на вектори — запускає AI-модель

Інтеграція (Integration)

MCP SDKБібліотека стандартного протоколу — дозволяє AI-помічникам підключатися
AxiosHTTP-клієнт — виконує мережеві запити до інших сервісів
LangChainВикористовується лише для розділення тексту — нічого важкого

НЕ використовується

torch · transformers · langchain-core
redis · fastapi · hnswlib

Цих важких бібліотек свідомо уникали заради швидкості та простоти.

Ми навмисно не використовуємо важкі бібліотеки — система легка та працює без потужних серверів

✅ Усі npm-залежності — чистий JavaScript, жодних вбудованих модулів. Працює на Windows, macOS і Linux без змін.

Запуск та встановлення

Три способи запуску Context Manager, залежно від ваших потреб.

Docker Compose (3 сервіси)

Docker (платформа контейнеризації) — це ніби "віртуальна коробка", яка містить усе необхідне для роботи. автоматичне керування та старт

context-manager:3847~43 MiBГоловний сервер
tei-embeddings:8080~107 MiBAI-модель для перетворення тексту на сенс
cm-mcp-adapter:8770~2.6 MiBMCP HTTP-адаптер

+ зовнішні: PostgreSQL :5432, Qdrant :6333

Docker — технологія "контейнеризації": програма пакується з усім необхідним для роботи, як консервна банка

Нативно (без Docker)

Встановлення безпосередньо на комп'ютер — без віртуалізації.

npm installВстановити всі залежності
npm run buildСкомпілювати TypeScript у JavaScript
npm startЗапустити сервер
npm run devЗапуск у режимі розробки

Потрібно: Node.js 18+, PostgreSQL, Qdrant

Керування процесами

Інструменти для автоматичного підтримання сервера в робочому стані:

Windowsnssm — запуск як служби Windows (автозапуск)
Linuxsystemd — запуск як служби Linux (автозапуск)
macOSlaunchd plist / Docker Desktop
"Служба" означає, що програма запускається автоматично під час завантаження комп'ютера та працює у фоновому режимі
