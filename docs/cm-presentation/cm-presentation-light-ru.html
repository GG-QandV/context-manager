001

Что такое Context Manager?

Инструмент, который дает AI-помощникам память — словно записная книжка, куда они записывают то, что узнали, а затем могут найти это снова.

Проблема

Каждый раз, когда вы говорите с AI, он начинает с нуля. Он не помнит, что вы обсуждали вчера, на прошлой неделе — даже пять минут назад. Весь контекст теряется.

Каждый разговор с AI начинается "с чистого листа" — предыдущий контекст теряется

Решение

Context Manager сохраняет разговоры двумя способами: как обычный текст в базе данных (PostgreSQL), и как "цифровые отпечатки смысла" — числа, которые показывают, о чем именно шла речь.

Сохранение в двух форматах: текст (для точного поиска) и "цифровой отпечаток смысла" (для поиска по похожим темам)

Результат

AI-помощники помнят прошлые разговоры, находят нужную информацию по смыслу и могут обмениваться контекстом между собой — без участия человека.

AI-помощники помнят предыдущие разговоры и могут обмениваться информацией друг с другом
Важное пояснение: Context Manager использует стандартный способ подключения под названием MCP. Представьте MCP как универсальный разъем (стандарт подключения) — так же как любые наушники с 3.5mm jack подходят к любому телефону, любой AI-помощник, который поддерживает MCP, может подключиться к Context Manager. Если ваш AI не поддерживает MCP, Context Manager также работает через обычный веб-адрес (HTTP — протокол передачи данных, по которому работают сайты).
002

Как это работает

Система имеет три логических слоя. Информация движется сверху вниз и в обратном направлении.

Как читать эту схему: Информация движется по шагам (1 → 2 → 3). Слой 1 — инструменты AI, которые вы используете (Antigravity, VS Code и т.д.). Слой 2 — "мозг" Context Manager, который обрабатывает запросы. Слой 3 — место, где живут данные: • PostgreSQL (реляционная БД — данные в таблицах) — сохраняет разговор как текст. ONNX Embedder — конвертирует текст в математический "отпечаток • Qdrant (векторная БД для поиска по смыслу) — сохраняет эти "отпечатки смысла" для быстрого поиска по смыслу. Ключевой момент: обе базы данных синхронизированы. Когда вы сохраняете разговор, текст попадает в PostgreSQL, затем ONNX Embedder конвертирует его в вектор и сохраняет в Qdrant автоматически. Обе базы всегда содержат одни и те же данные — просто в разных форматах (текст vs цифровой отпечаток смысла).
003

Возможности

Context Manager предлагает несколько инструментов — каждый выполняет одну конкретную задачу, словно кнопки на пульте дистанционного управления.

Сохранение контекста

Три уровня детализации, в зависимости от того, сколько вы хотите сохранить:

✓
Brief (кратко) — автоматическая сводка разговора на 200-300 символов. Словно твит о том, что произошло.
✓
Important (важное) — структурированные заметки с темами, до 3 000 символов. Словно итог совещания.
✓
Full (полностью) — полный лог разговора дословно. Словно сохранить всю запись совещания.

Поиск и анализ

Находите прошлые разговоры разными способами:

✓
Semantic search (поиск по смыслу) — находит разговоры по смыслу, даже если вы не помните точных слов. Например, поиск "проблемы с сервером" найдет разговоры об "ошибках подключения к базе данных".
✓
Filtered query (фильтр по параметрам) — поиск с фильтрами: по дате, AI-агенту, режиму сохранения.
✓
Cross-agent search (поиск между агентами) — один AI-помощник может найти то, что записал другой AI-помощник.

Мониторинг

Просмотр того, что находится в системе памяти:

✓
Agent list (список агентов) — показывает всех AI-помощников и количество сохраненных ими записей.
✓
Statistics (статистика) — детали о конкретном помощнике или сессии.
✓
Export (экспорт) — выгрузка сессии в JSON (текстовый формат, понятный программам).

Встроенные помощники

Умные инструменты, которые работают автоматически:

✓
Content Detector (определение типа содержимого) — автоматически распознает, что именно читает: код, команды, таблицы или ошибки — и обрабатывает соответственно.
✓
Time Parser (распознавание дат) — понимает фразы "вчера", "последние 3 дня" на русском и английском.
✓
Chunking (разделение текста на части) — разбивает длинные тексты на меньшие куски для лучшего качества поиска.

Способы подключения

Три способа подключить Context Manager к вашим AI-инструментам:

✓
Standard MCP (стандартный) — для AI-програм на вашем компьютере. Самый распространенный способ.
✓
Web MCP (через интернет) — для AI-програм в браузере. Использует порт :8770.
✓
Direct HTTP (прямой доступ) — без MCP, через веб-запросы на порт :3847. Если ваш AI не поддерживает MCP.
Что такое MCP? MCP — это единый стандарт подключения (язык общения) для AI-помощников. Представьте, что у каждого AI-помощника есть "вилка", а у Context Manager — "розетка". MCP — это международный стандарт, которого придерживаются все производители, чтобы вилка подошла к розетке. Благодаря MCP вам не нужно настраивать каждую программу отдельно — если AI поддерживает MCP, он автоматически "видит" Context Manager и может в него писать и читать. Альтернатива (если ваш AI не поддерживает MCP): Context Manager также работает через HTTP (протокол передачи данных — тот же, по которому работают веб-сайты). В этом случае подключение настраивается один раз через веб-адрес (URL) и пароль.

Технологии

Современные, легкие технологии. Никаких тяжелых фреймворков — работает офлайн, на любом компьютере.

Среда выполнения (Runtime)

Node.jsСреда для JavaScript — движок, который запускает сервер
TypeScriptЯзык программирования — JavaScript с типами для надежности
FastifyВеб-фреймворк — обрабатывает HTTP-запросы (интернет-обращения)
PinoЖурнал событий (логгер) — записывает, что делает система

Хранилища (Storage)

PostgreSQLРеляционная БД (база данных) — хранит текст с индексами для поиска
QdrantВекторная БД — хранит "отпечатки смысла" для поиска по смыслу
ONNX EmbedderONNX runtime — запускает модель multilingual-e5-small

Интеграция (Integration)

MCP SDKБиблиотека стандартного протокола — позволяет AI-помощникам подключаться
AxiosHTTP-клиент — выполняет сетевые запросы к другим сервисам
LangChainИспользуется только для разделения текста — ничего тяжелого

НЕ используется

torch · transformers · langchain-core
redis · fastapi · hnswlib

Этих тяжелых библиотек сознательно избегали ради скорости и простоты.

Мы намеренно не используем тяжелые библиотеки — система легкая и работает без мощных серверов

✅ Все npm-зависимости — чистый JavaScript, никаких встроенных модулей. Работает на Windows, macOS и Linux без изменений.

Запуск и установка

Три способа запуска Context Manager, в зависимости от ваших потребностей.

Docker Compose (3 сервиса)

Docker (платформа контейнеризации) — это словно "виртуальная коробка", которая содержит все необходимое для работы. автоматическое управление и старт

context-manager:3847~43 MiBГлавный сервер
tei-embeddings:8080~107 MiBAI-модель для преобразования текста в смысл
cm-mcp-adapter:8770~2.6 MiBMCP HTTP-адаптер

+ внешние: PostgreSQL :5432, Qdrant :6334

Docker — технология "контейнеризации": программа упаковывается со всем необходимым для работы, как консервная банка

Нативно (без Docker)

Установка непосредственно на компьютер — без виртуализации.

npm installУстановить все зависимости
npm run buildСкомпилировать TypeScript в JavaScript
npm startЗапустить сервер
npm run devЗапуск в режиме разработки

Требуется: Node.js 18+, PostgreSQL, Qdrant

Инструменты для автоматического поддержания сервера в рабочем состоянии:
